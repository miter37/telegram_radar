Market Radar Desktop

텔레그램 주식 피드를 실시간 수집해 LLM 한 번의 호출로 주제, 주요내용, 정규화 태그, 중요도 점수, 관심분야 연관성 점수를 생성하고, DB에 저장한 뒤 태그·주제별 흐름을 추적하는 윈도우 데스크톱 앱.

서비스 구조
1Telegram ingest

채널명, 시간, 원문, 메시지 링크를 수집하고 원문 테이블에 저장.

2LLM one-pass

원문 1건 또는 소량 배치를 JSON으로 구조화. 별도 맥락 입력 없음.

3Validate

주제 12자, 주요내용 50자, 점수 0~100, JSON 필수값 검사.

4Normalize

태그명은 사전/룰/유사도 기반으로 기업·인물·산업 표준값에 매핑.

5Store

feed_items, feed_signals, normalized_tags, topic_clusters에 저장.

6Analyze flow

일자별 주제, 태그 상승률, 관심도 변화, 분석 요청 결과 생성.

실시간 저장 단계의 LLM 입력은 단순하게 유지한다. 메시지 원문, 날짜, 채널명, 메시지 링크, 사용자 관심분야만 넣고, 최근 상위 주제나 대량 태그 사전은 프롬프트에 넣지 않는다.
LLM 입력/출력 설계

입력

datetime수집 시각
channel_name텔레그램 채널명
message_text피드 원문
message_url원문 이동 링크. 없으면 null.
user_interests관심 종목/산업/키워드의 짧은 리스트

출력

topic주제. 12자 이내.
main_content주요내용. 50자 이내.
tags정규화 이름으로 쓰려는 태그 리스트.
tag_groupscompanies, people, industries로 분리된 태그 묶음.
importance_score중요도 0~100.
interest_score관심분야 연관성 0~100.
LLM 프롬프트 초안
{SYSTEM}
너는 주식 관련 텔레그램 피드를 DB에 저장 가능한 구조로 변환하는 정보 추출 엔진이다.
반드시 JSON만 출력한다. 설명문, markdown, 주석은 출력하지 않는다.

원칙:
1. 원문에 없는 사실을 만들지 않는다.
2. 투자 의견, 매수/매도 추천을 쓰지 않는다.
3. topic은 12자 이내 한국어 명사구로 쓴다.
4. main_content는 50자 이내 한국어 한 문장으로 쓴다.
5. tags는 기업, 주요 인물, 산업분야, 제품/기술 중심으로 뽑는다.
6. tags는 가능한 표준명으로 쓴다. 예: 하닉 → SK하이닉스, 젠슨 → 젠슨 황.
7. 확신이 낮아도 별도 후보/타입을 만들지 말고, 가장 그럴듯한 표준 태그명만 tags에 넣는다.
8. tag_groups에는 tags를 companies, people, industries로만 분리한다.
9. 해당 그룹이 없으면 빈 배열을 쓴다.
10. importance_score와 interest_score는 0~100 정수로 쓴다.
11. importance_score는 시장 민감도, 실적/수급/규제/공급망 관련성, 구체성, 즉시성 기준으로 판단한다.
12. interest_score는 user_interests와의 직접/간접 관련성 기준으로 판단한다.

{INPUT}
{
  "datetime": "{{datetime}}",
  "channel_name": "{{channel_name}}",
  "message_text": "{{message_text}}",
  "message_url": "{{message_url}}",
  "user_interests": {{user_interests_json}}
}

{OUTPUT_JSON_SCHEMA}
{
  "datetime": "YYYY-MM-DDTHH:mm:ss+09:00",
  "channel_name": "string",
  "topic": "string, max 12 Korean chars",
  "main_content": "string, max 50 Korean chars",
  "tags": ["string"],
  "tag_groups": {
    "companies": ["string"],
    "people": ["string"],
    "industries": ["string"]
  },
  "importance_score": 0,
  "interest_score": 0,
  "should_alert": false
}
태그 정규화는 LLM 출력 이후 시스템 레이어에서 다시 수행한다. LLM이 뽑은 tags는 초안값이고, DB에는 canonical_tag_id까지 매핑된 값을 저장한다.
DB 설계
테이블필드역할
feed_itemsid, datetime, channel_name, message_text, message_url, raw_hash, collected_at원문 저장. 실시간 피드에서 원문 팝업을 열 때 사용.
llm_extractionsfeed_id, prompt_version, raw_json, parsed_ok, error_messageLLM 원본 결과 보관. 프롬프트 개선과 재처리 추적.
feed_signalsfeed_id, date, channel_name, topic, main_content, importance_score, interest_score, should_alert실시간 DB 피드 화면의 메인 테이블.
signal_tagsfeed_id, canonical_tag_id, canonical_name, tag_group, normalize_confidence피드별 표준 태그 연결. tag_group은 company, person, industry.
canonical_tagstag_id, canonical_name, tag_group, aliases, ticker, sector, parent_tag_id표준 태그 사전. 운영자가 수정 가능.
topic_clusterstopic_id, topic, canonical_tags, first_seen_at, last_seen_at, feed_count, cluster_score유사 피드 묶음. 중복 제거와 일자별 중요 주제 계산.
daily_topicsdate, topic_id, daily_rank, summary, representative_feed_ids, total_score날짜별 핵심 주제 화면에 사용.
tag_flow_metricstag_id, date, feed_count, avg_importance, avg_interest, velocity, acceleration태그별 흐름 대시보드와 과거 대비 변화 분석.
